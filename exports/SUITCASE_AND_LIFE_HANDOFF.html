2. Product Overview & Personas

The product

A single-page editorial website that acts as a public "display shelf" for the curators' personal collection of objects (electronics, furniture, ceramics, cameras, etc.). The data lives entirely in a Google Sheet so the curators never have to touch code to add a new item. The site auto-syncs every 5 min.

Personas

PersonaJob-to-be-doneKey surface
Curators (Ankeet & Sruthi) Add an object: photos, story, price, source, tags. Want zero friction. Never want to deploy code to add an item. The Google Sheet (single source of truth)
Visitors (Instagram followers, friends, design lovers) Browse the shelf, filter by category, read the story behind each piece, mark "Add to Shelf" for things they love. Home (Hero → Bento → Archive), product modal, /my-shelf

Three sections (Home page)

  1. Cinematic Hero with a blurred lifestyle background, oversized brand wordmark, and updated-weekly meta.
  2. Bento Grid with 4 featured tiles + 1 mandatory black "Archive Note" accent card. Featured items are pulled from sheet flag; if <4 featured, top-liked items fill the rest.
  3. Full Archive — sticky category filter pills, then a responsive grid of ProductCards. 2 cols on mobile, 3 on tablet, 4 on desktop.

5. Backend API Reference

MethodPathPurposeResponse shape
GET/api/products All products from the cached Sheet (or sample fallback) merged with MongoDB like counts. Product[] (see §6)
GET/api/categories Distinct categories present in the sheet, alphabetised. string[]
POST/api/products/{slug}/like Increment the visitor "Add to Shelf" counter in MongoDB. { id: string, likes: number }
GET/api/refresh Force-clear the Sheet cache and re-fetch immediately. { ok: true, count: number }

All endpoints are public (no auth). CORS allowed from CORS_ORIGINS env var.

7. Frontend Component Reference

App.js — root

  • Sets up React Router with /, /my-shelf (alias /my-list), /about.
  • Loads products once on mount; tracks activeId for the modal.
  • Owns the scroll-lock via savedScrollRef + useEffect on !!activeId.
  • Owns the likedSet (Set of liked IDs from localStorage[sl_liked_v1]).

components/ProductDetail.jsx

The most intricate component. Two layouts coexist in the DOM:

  • Mobile (md:hidden): full-screen, sticky close button at fixed top-3 right-3 z-30, scrollable content area, sticky bottom action bar (Add to Shelf + open-link arrow). Gallery has onTouchStart/onTouchEnd handlers — horizontal swipe >40px advances/rewinds.
  • Desktop (hidden md:grid): 7/5 column grid, fixed top (category + close), title block, scrollable description, fixed footer (meta, tags, actions, prev/next chevrons).

Dialog uses modal={false} so Radix doesn't run its own scroll-lock; we render our own backdrop at z-99 (data-testid="modal-backdrop") above the header (z-70) but below the dialog content (z-101).

components/Header.jsx

  • fixed top-0 z-[70]. Goes transparent over the Hero on Home, opaque otherwise.
  • Mobile hamburger opens a full-screen z-65 menu panel.
  • Brand mark mixes Bricolage Grotesque + EB Garamond italic ampersand.

pages/MyList.jsx (route /my-shelf)

  • Filters all products by likedSet.
  • Empty state: inline EmptyShelfDog SVG (line-art bookshelf + confused terrier with question mark) + CTA Add to Shelf linking to /#archive.

9. Known Quirks & Hard-Won Lessons

Modal scroll-lock — the single most fragile thing in this codebase. Several iterations failed before this final pattern landed. Do not refactor casually.

The pattern (in App.js)

// Capture BEFORE setActiveId — Radix mount auto-scrolls so reading
// window.scrollY inside the useEffect is too late.
const savedScrollRef = useRef(0);
const openProduct = (p) => {
  savedScrollRef.current = window.scrollY;
  setActiveId(p.id);
};

// Effect depends on !!activeId so prev/next switches don't unlock.
useEffect(() => {
  if (!isOpen) return;
  const savedY = savedScrollRef.current;
  body.style.position = "fixed";
  body.style.top = `-${savedY}px`;
  body.style.left = "0"; body.style.right = "0"; body.style.width = "100%";
  return () => {
    body.style.position = ""; body.style.top = "";
    body.style.left = ""; body.style.right = ""; body.style.width = "";
    // OVERRIDE scroll-behavior:smooth from index.css to avoid 600ms anim.
    html.style.scrollBehavior = "auto";
    window.scrollTo(0, savedY);
    html.style.scrollBehavior = "";
  };
}, [isOpen]);

Why each detail matters

  • Capture in click handler, not in effect — Radix Dialog's mount triggers an auto-scroll up to thousands of px. Reading window.scrollY inside the effect captures the wrong value. Iteration 3 of testing found this clearly.
  • position:fixed with top:-savedY — only pattern that truly freezes the visual position regardless of focus events.
  • Override scroll-behavior:smooth on cleanup — index.css line 40 sets this globally, which would animate the post-close scrollTo over ~600ms. Override to auto for an instant snap.
  • modal={false} on Radix Dialog — using modal=true activated react-remove-scroll which conflicted with our custom lock and auto-scrolled the page. We render a custom backdrop instead.
  • Custom backdrop at z-99 — the header is z-70, dialog content is z-101. Without a backdrop covering z-71→100, the header is visible above the modal. The custom backdrop hides it.

Other gotchas

  • html, body { min-height: 100% } in index.css — was previously height: 100%, which caused scrollY to reset when react-remove-scroll set overflow:hidden. Switched to min-height.
  • Header watches scrollY > 40 to switch from transparent → opaque. While the modal is locked, scrollY is 0, so the header would normally go transparent. We don't fix this because the backdrop covers it anyway. If you ever drop the backdrop, you'll need to force the "scrolled" state during modal open.
  • Header.jsx uses document.body.style.overflow = 'hidden' for its own mobile menu lock. This is fine on its own but can clash with the modal lock if both opened simultaneously — they don't, but worth knowing.
  • Both desktop and mobile modal trees are in the DOM (toggled by md:hidden CSS). Test IDs like btn-interested exist in the mobile tree, btn-interested-desktop in the desktop tree. When writing tests, query by visibility or by viewport-specific ID.

10. Local Development & Deployment

Frontend

cd /app/frontend
yarn install
yarn start         # http://localhost:3000

Backend

cd /app/backend
pip install -r requirements.txt
uvicorn server:app --host 0.0.0.0 --port 8001 --reload

MongoDB

Local instance, no auth. Connection via MONGO_URL. The likes collection is auto-created on first POST.

backend/.envSHEET_CACHE_TTLSeconds, default 300

Deployment

Currently runs on the Emergent platform managed by supervisor. The frontend is built with yarn build and served as static assets; the backend is run via uvicorn. Any container platform (Render, Railway, Fly, Vercel + Render) works as long as the two services can talk to each other and the frontend is given the backend URL via REACT_APP_BACKEND_URL.

  • Lenis momentum scrolling.
  • View Transitions API for card → modal shared-element animation.
  • Keyboard arrow keys inside the gallery (currently only switches products).
  • "Liked by you" sub-shelf at top of archive when likedSet.size >= 1.

P3 — Growth

  • Email capture for "tell me when new things are added" — would require a backend list + a transactional email integration.

  • Shareable "My Shelf" — encode the liked-IDs into a URL hash (or a tiny share token) so visitors can DM their curated mini-shelf to friends.

Phase 4 — Modal scroll-lock saga

This took several iterations and is the most subtle work in the codebase. Key data points captured by the testing agent across iterations:

  • Iter 1: opening the modal jumped the background to Hero.
  • Iter 2: initial fix scrolled-to-top because html { height:100% } + react-remove-scroll caused scrollY to reset.
  • Iter 3: capturing scrollY inside the useEffect read the wrong value (Radix mount auto-scrolled before the effect ran).
  • Iter 4: Final fix — capture in the click handler via a useRef, plus override scroll-behavior:smooth on cleanup. 100% pass.

Phase 5 — Final UX pass (current)

  • Modal layered properly with a custom z-99 backdrop covering the navigation.
  • Mobile modal redesigned: scrollable content, sticky close at top-right, sticky bottom action bar, swipe-to-change image.
  • Renamed I'm interested → Add to Shelf; toggled to On your shelf after liking.
  • Renamed My list → My shelf, route /my-shelf, with a custom inline-SVG empty state (line-art bookshelf + confused dog).
  • Bonus: fixed fetchpriority → fetchPriority React warning in Hero.jsx.
